项目概览

该项目并非简单 FPS 样例堆叠，而是在现有射击框架基础上，扩展出 Roguelite 单局玩法的核心系统： 关卡推进、波次刷怪、奖励选择、数值成长以及最终 Boss 结算。

玩法闭环

玩家在多个关卡段之间推进，完成当前房间的敌人波次后解锁出口， 随后触发三选一卡奖励，逐步塑造局内 build，并在最终进入 Boss 房间完成一局挑战。

数据驱动成长

奖励词条基于稀有度、权重和局内去重规则进行抽取， 可对玩家与武器的伤害、暴击、弹道数量、反弹等属性产生即时影响， 让不同局的战斗体验保持差异化。

技术实现细节

这个项目的实现重点主要集中在关卡流程、战斗成长、表现反馈和运行时性能几个方向。

段式关卡与运行流程

关卡生成器负责组织固定首段、随机普通段与 Boss 段， 支持运行时追加、预加载和推进切换，并结合房间门禁实现清怪后开门、进入下一段的节奏编排。

  • 普通段随机串联，Boss 段固定收尾
  • 支持预加载与运行时追加下一段
  • 推进时同步处理碎片场景与当前段上下文

三选一卡与词条系统

奖励管理器会在房间完成后构建候选词条池， 根据稀有度权重和进度曲线动态抽取奖励，并将选择结果即时应用到运行时属性系统中。

  • 支持动态稀有度提升与局内唯一词条去重
  • 覆盖玩家属性和武器属性两类强化目标
  • 面向 Roguelite build 构筑做即时数值反馈

Game Feel 与表现强化

项目专门实现了镜头呼吸、包络震动、位置冲量和顿帧反馈， 并把 DOTween 用于门禁开关和奖励卡牌 UI 动效，强调开火、命中、击杀、清房与选卡的反馈层次。

  • 基于 Perlin Noise 的多轴屏幕震动
  • 支持多种震动包络曲线与位置冲量
  • 卡牌弹出、选中和门开关均有弹性动效

性能与工程组织

项目在玩法扩展的同时保留了 Unity 实战开发中的性能意识与模块边界， 对高频对象和导航系统进行了专门处理。

  • 对象池用于敌人、弹丸与特效复用
  • NavMeshSurface 支持防抖与异步重建策略
  • 代码按 Game、Gameplay、AI、UI、Roguelike 分模块组织

项目总结

项目简介

这是一个基于 Unity 2022.3、C# 和 URP 开发的第一人称 Roguelite 射击项目， 我围绕段式关卡推进、波次战斗、三选一卡、词条成长与 Boss 结算搭建了完整的单局玩法闭环。

核心实现

  • 实现段式关卡生成、预加载与运行时推进切换
  • 完成奖励词条系统与玩家、武器属性聚合
  • 结合对象池与 NavMesh 重建优化高频战斗场景
  • 通过镜头震动、顿帧和 UI 动效强化战斗反馈
